# CLAUDE.md

This repository implements CSC 580 Final Project Option 1: **"Dream the Road — Tiny Dreamer for Highway-env"**.

## Project Goal
Build a Dreamer V1-style world model system for Gymnasium Highway-Env that performs highway driving as fast as possible while avoiding collisions.

The target task is the `highway-v0` environment, whose objective is to reach high speed while avoiding collisions with neighboring vehicles.

## Assignment Constraints
- Use **Gymnasium Highway-Env**
- Use the **`highway-v0`** environment
- Initialize the environment with **`render_mode="rgb_array"`**
- Use **continuous actions** (two continuous values)
- Use **4 lanes or more** (the default is 3)

## Required System Structure
The implementation must include two main parts:

### 1) World Dynamics Model
At minimum, include:
- **Encoder**
- **RSSM** (Recurrent State-Space Model)
- **Reward model**
- **Decoder**

This part is trained using image/frame transitions collected from actual environment interactions. Transitions are sampled from a replay buffer. The assignment describes this part as supervised learning over ground-truth transitions.

### 2) Actor-Critic Model
Include:
- **Actor / Policy model**
- **Critic / Value model**

This part is trained using imagined trajectories generated by the learned world dynamics model. The assignment describes the learning here as reinforcement learning using temporal-difference style updates.

## Required Training Workflow
1. Configure Gymnasium + Highway-Env for continuous control.
2. Collect random transitions in the form  
   `(frame/image, action, reward, next frame/image)`  
   into a replay buffer as a warm start.
3. Train the world/environment dynamics model by repeatedly sampling from the replay buffer.
4. Train the actor-critic model using rollouts/trajectories generated with the learned environment dynamics model.
5. Act in the real environment using the learned models and store additional transitions in the replay buffer.
6. Repeat steps 3–5 for multiple iterations.

## Required Evaluation
- From a given start state, generate an imagined trajectory of **K** steps.
- Measure loss values in some appropriate way.
- Devise quantitative metrics to evaluate the learned model.
- If feasible, optionally compare the imagined trajectory with the trajectory from the original / kinematics model.

## Required Visualization
- Visualize the **n-step predictions** made by the world model versus the **actual results** from the training set.
- Create a few **sample visualizations and videos** for submission.

## Implementation Flexibility
- You may implement your own Dreamer-style system from scratch.
- You may also adapt an existing Dreamer-style codebase/framework, provided the final implementation satisfies the assignment requirements.
